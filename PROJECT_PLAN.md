# Project Plan

## 1. 项目最终目标

CompCheck（成分小查）的目标是完成一个可正式使用、可持续迭代的成分查询与分析 App。

核心能力包括：

- 成分搜索
- 成分详情
- 成分表识别
- 成分风险提示
- 收藏与历史
- 移动端友好页面
- 后续 AI 分析能力扩展
- 品牌名称：CompCheck
- 中文名称：成分小查

## 2. 当前阶段

当前阶段：第六阶段 / AI 与 OCR 扩展预留

说明：第一到第四阶段的基础闭环已实现；第五阶段已完成基础移动端和视觉统一；第六阶段已完成前端接口预留和未配置降级，但真实 AI/OCR 仍需要后端代理。

## 4. 已完成功能

- 项目基础目录结构：`src/components`、`src/pages`、`src/router`、`src/services`、`src/store`、`src/data`、`src/types`、`src/utils`。
- 本地开发服务器和构建脚本。
- 首页快速入口和移动端优先布局。
- 成分搜索页和无结果提示。
- 成分详情页和谨慎风险说明。
- 成分表文本输入分析。
- 本地成分库和成分服务层。
- 收藏、取消收藏、搜索历史、清空历史和本地持久化。
- AI 分析 service 预留，未配置时返回明确降级信息。
- OCR service 预留，图片上传入口不伪造识别结果。
- `.env.example`、`COMMANDS.md`、`AI_REVIEW.md`。
- 基础 lint、test、build、启动验证流程。
- 品牌名已统一为 CompCheck（成分小查）。

## 5. 未完成功能

- 真实 OCR 图片识别。
- 真实 AI 分析。
- 服务端代理和密钥管理。
- 更完整的成分库数据来源与维护流程。
- 端到端自动化页面测试。
- 真实移动设备或浏览器截图验收。
- 生产部署方案。

## 6. 当前阻塞问题

- 真实 AI/OCR 需要后端代理、供应商选择和密钥配置，不能在前端直接写死或暴露 API Key。
- 当前成分数据为本地维护的小型数据集，正式使用前需要扩充数据来源和审核流程。
