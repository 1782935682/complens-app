## 0. 适用范围与角色

- 适用产品：CompLens/成分镜 —— 食品配料表拍照识别（OCR）与成分分析 App。

## 1. 图片与 OCR 隐私

### 1.1 图片用途声明（最小必要）

- 用户上传/拍摄的图片**仅用于 OCR 文字识别与随后的成分分析**，不用于其他目的。
- 不得将图片用于训练模型、广告画像或与上述无关的处理（除非另行获得明示同意，且需「待法务确认」）。

### 1.2 图片存储真实口径

- 图片存储于设备本地 **IndexedDB**（库/表：`compcheck-images` / `scan-images`），用于历史记录与档案功能。
- `localStorage` **仅存元数据/索引/设置**，不存图片二进制。
- 前端**不暴露** OCR / AI 的 API Key；OCR 调用一律经由后端代理。

### 1.3 是否长期保存原图

- 当前默认：图片保存在**本机 IndexedDB**，用于历史/档案展示。
- **是否长期保存原图、保留时长** = **待确认 / 待法务确认**（需用户拍板：是仅保存识别文本还是同时保留原图，以及本机保留周期）。
- **服务端是否留存原图** = **待确认 / 待法务确认**（当前口径为后端仅做 OCR 代理转发，不确认是否落盘；须在隐私政策中明确声明）。

### 1.4 OCR 临时图片删除策略

- 原则：用于一次性识别的**临时图片应在识别完成后尽快删除**，不做无目的留存。
- 后端代理在转发给 OCR 服务后，应避免持久化原图；如需缓存，须设定明确的过期与清理策略。**具体实现状态 = 待确认**。

### 1.5 第三方 OCR 与数据流向声明

- **不得把图片传给任何未在隐私政策中声明的第三方服务。**
- 当前（本机/开发）：使用**本机 RapidOCR**（`/home/downloads/tools/complens-ocr`，当前代码默认 `http://127.0.0.1:8000`），由 CompCheck 后端代理调用（`OCR_PROVIDER=rapidocr`、`OCR_SERVICE_URL`）；OCR 服务**不直接暴露公网**。
- 统一架构目标：本机 OCR 服务仅监听内网/本机，目标地址 `http://127.0.0.1:18080/ocr`，由后端通过 `OCR_LOCAL_URL` 调用；当前变量和端口迁移未完成前，不得让前端或 App 直连该服务。
- 生产计划：切换至 **Aliyun（阿里云）OCR**。一旦切换，必须：
  1. 在隐私政策中明确声明「图片将传输至阿里云 OCR 服务进行识别」；
  2. 与供应商签署数据处理协议（DPA）—— **待法务确认**；
  3. 说明传输与处理的安全措施、留存与删除约定。

## 2. 权限说明

封装框架：Capacitor 7.x。原则：**即用即申请（按需触发）**，不在启动时一次性索取全部权限；每项权限均向用户说明用途。

| 权限 | 插件/来源 | 用途 | 申请时机 | 状态 |
| --- | --- | --- | --- | --- |
| 相机 | `@capacitor/camera` | 拍摄食品配料表用于 OCR 识别 | 用户首次点击「拍照」时申请 | 已规划 |
| 相册/照片库 | `@capacitor/camera` | 从相册选择食品包装图片 | 用户首次点击「从相册选择」时申请 | 已规划 |
| 网络 | 系统/网络栈 | 调用后端 API（成分库、OCR 代理、登录云同步） | 应用使用网络功能时 | 默认需要 |
| 本地存储 | IndexedDB / localStorage | 保存历史、档案、设置与图片（见第 1 节） | 本地读写时（通常无需系统弹窗） | 已使用 |
| 通知 | 系统通知 | 提醒类功能 | **后置**，待相关功能上线后即用即申请 | 后置 |
| 定位 | —— | **默认不需要** | 不申请 | 默认不需要 |

- iOS 权限描述文案见 [`docs/ios-plist-additions.md`](../ios-plist-additions.md)：
  - `NSCameraUsageDescription = "用于扫描食品成分表"`
  - `NSPhotoLibraryUsageDescription = "用于从相册选择食品包装图片"`
- Android 权限清单（`AndroidManifest.xml`）需与上述用途一致 = **待确认**（以实际工程为准）。

## 3. 合规文案规范

面向食品标签解读和消费决策的表述需克制、不绝对、不构成医疗或营养诊断。

### 3.1 禁止用语清单

以下表述**禁止使用**：

- 绝对安全 / 绝对有害
- 可以买 / 不能买
- 健康 / 不健康
- 安全 / 有害
- 致癌
- 治疗 / 诊断
- 一定不能吃
- 一定过敏
- 有毒

（即：任何「绝对化」「医疗化」「诊断化」或替用户下购买命令的断言。）

### 3.2 推荐用语清单

以下为**推荐替代表述**：

- 建议关注
- 部分人群可能需要留意
- 仅供配料信息参考
- 不构成医疗建议
- 具体情况请结合个人体质或咨询专业人士

### 3.3 适用范围

- 适用于成分分析结论、标签、提示语、风险提示、推送文案与营销素材。
- 内容来源与纠错口径参见 `DATA_TRUST_SPEC.md` 及 `legalContent.js` 的「内容来源与审核说明」。
