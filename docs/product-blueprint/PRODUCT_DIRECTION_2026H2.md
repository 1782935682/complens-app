## 0. 为什么要修订

项目定位正确，但**资源投放重心错位**。约 60% 的工程精力压在 **GB2760 全量法规数据治理**（264 页 PDF、数千条 staging、7 级可信状态、admin 后台、人工复核流水线）上——这是监管 / 营养师级别的精度需求，对**普通大众的边际价值很低**，却又最重、最难变现、最卷不过专业团队。

对一个个人开发者，这是最危险的错配：把稀缺精力投在"权威成分百科"上，却把真正的差异化（**个性化关注 + 大白话翻译 + 比一比**）和分发（**上线触达真实用户**）一再后置。

本次修订的核心动作：**把重心从"数据有多全"转回"普通人有没有看懂、用没用起来"。**

---

## 1. 产品定位（收敛后）

> **「拍一下，看懂配料表」——帮普通人把看不懂的食品标签，翻译成"对我要不要紧"。**

- 不是成分百科，不是法规查询系统，不是营养诊断工具。
- 核心能力是**翻译 + 个性化提醒**，不是**裁决**。
- 守住合规边界（沿用 `CONSUMER_DECISION_SPEC.md` 文案约束）：只说"建议关注"，不说"健康 / 能买 / 有害"。

一句话差异化：
> 别人告诉你"这个成分是什么"，成分镜告诉你"**这个东西，对你 / 对你家孩子，要不要紧**"。

---

## 2. 目标用户（聚焦三类高频人群）

聚焦**高频、有情绪、愿意复用**的人群，其余作为长尾流量，不为其改变路线。

| 人群 | 真实场景 | 复用 / 付费潜力 |
|---|---|---|
| **新手父母**（最高价值） | 给孩子买零食 / 辅食，怕添加剂、香精色素、高糖高钠 | 几乎每次买零食都查，长期复用、最愿付费 |
| **控糖 / 减脂人群** | 看"0 蔗糖"是不是真无糖、代糖是啥、碳水高不高 | 日常高频，强刚需 |
| **过敏 / 忌口人群** | 快速扫有没有自己过敏 / 忌口成分 | 错一次代价大，依赖性强 |

**明确不服务**：营养师、医生、食品从业者、监管。他们要的精度，本产品不提供，也不该提供。

## 3. 核心功能与优先级

| 功能 | 用户价值 | 难度 | 优先级 |
|---|---|---|---|
| 拍照 → 清晰识别自动生成报告；低置信 / 失败时文本确认补充（主路径） | 高 | 中 | **P0** |
| **添加剂大白话翻译**（精选词表，约 150–300 条常见添加剂：归类 + 一句人话作用 + 是否常见） | 高 | 低-中 | **P0** |
| 个性化关注项（控糖 / 儿童 / 过敏 / 减脂） | 高 | 低 | **P0** |
| 历史 / 收藏 | 中-高 | 低 | **P0** |
| 比一比（双品对比，按关注目标） | 高 | 中 | **P1** |
| 报告分享摘要（拉新引擎） | 中 | 低 | **P1** |
| 家庭多档案（不同家人不同关注项） | 中 | 低 | **P1** |
| 自定义关注 / 忌口黑名单 | 中 | 低 | **P1** |
| AI 兜底解释（生僻成分） | 中 | 中-高 | **P2** |

**关键点**：「添加剂大白话翻译」用一份**精选词表**即可覆盖约 90% 日常场景，**不需要 AI、不需要全量 GB2760**。已有的 GB2760 / 成分库成果，转为这份词表的数据来源即可，**停止继续投入全量治理**。

## 4. 三个版本的迭代路线

### 第一版（先活下来 / 验证需求）
- **收敛到微信小程序单端**（分发成本最低、最容易触达真实用户）。
- 跑通：拍照 → OCR / 商品身份识别 → 清晰识别自动生成大白话解读报告（含添加剂翻译）→ 历史；只有商品名、品牌、商品码或二维码时，先通过 AI 搜索公开标签线索生成带来源标注的参考报告，搜不到配料 / 营养时进入信息不足 / 手动补充。
- 关注项先做 **控糖 + 儿童 + 过敏** 三套。
- **OCR 保留自建 RapidOCR 路线**（用户端 → 后端 API → RapidOCR 服务，详见 `ARCHITECTURE_SPEC.md`）。
- 成功标准：**有 ≥100 个真实用户在用，看留存和被分享率**，而非数据覆盖率。

### 第二版（建立复用与差异化）
- 比一比（双品对比）。
- 自定义关注项 / 忌口黑名单。
- 家庭多档案。
- 报告分享摘要（拉新主引擎）。

### 第三版（深度与付费）
- 多平台扩展（H5 / App）——**仅在小程序验证成功后才做**。
- 历史趋势（"你最近买的零食添加剂偏多"）。
- 进阶人群关注模板（孕期、三高关注、健身增肌等）。
- AI 增强（生僻成分兜底解释）——锦上添花，不作地基。

## 6. 免费 vs 付费

现实判断：**大众工具型产品付费空间有限，频次是最大敌人**。能付费的是被"高频 + 强依赖"绑定的人群，因此付费绑定持续价值，而非锁基础功能。

**永久免费（保口碑与传播）**
- 拍照解读、添加剂翻译、基础关注项、最近 N 条历史、分享摘要。

**付费会员（绑定高频人群持续价值）**
- 无限历史 / 收藏档案
- 比一比（对比是决策刚需）
- 家庭多档案 + 自定义关注 / 黑名单
- 进阶人群关注模板
- 报告导出 / 无广告

**定价建议**：低价订阅（如 ¥12/月 或 ¥68/年）或一次性买断，主打新手父母与控糖人群。不做高价，大众盘子撑不起。

## 7. 最终产品形态与核心流程

**形态**：一个微信小程序（先），一句话定位"拍一下，看懂配料表"。

**首页 = 单一拍照入口**：首页只放正中央一个大拍照按钮 + 字标 + 标语，不堆其他元素，把用户唯一核心动作（拍）做到最聚焦。「我的关注」走底部 TabBar；「成分查询」作为辅助功能收进「我的」页，不在首页抢焦点。

```
① 拍 / 选 食品标签照片
      ↓
② OCR / 商品身份识别 → 清晰可生成时自动报告；低置信、失败、手动输入或信息不足时确认 / 补充
      ↓
③ 自动拆配料 + 解析营养表
      ↓
④ 结合「我的关注项」生成解读报告
   · 大白话翻译每个看不懂的成分
   · 命中我关注的点（糖 / 钠 / 过敏 / 儿童）高亮
   · 一句话建议（"建议重点看：糖、甜味剂"）
      ↓
⑤ 存历史 / 生成分享卡 / 加入比一比
```

## 8. 相比同类工具的独特价值

- **vs 通用搜索 / AI 拍照问**：它们一问一答、无记忆、不懂"我"；成分镜一次拍照即**结合个人关注项**出整理结论，可复用、可对比。
- **vs 专业成分 / 法规 App**：它们准但难懂、面向专业人群；成分镜面向看不懂的人，做"翻译"不做"裁决"。
- **vs 美妆成分 App（如美丽修行）**：那是美妆赛道；食品标签 + 个性化关注（尤其新手父母）是相对空白且高频的窄口。
